Experience

Senior Developer at BBC News

August 2014 — Present

I currently work for the core BBC News team. My focus recently has been on web page performance, and on building the deployment tooling and automated test suites around our new web application framework. I run the Automated Testing and DevOps working groups, which I use to drive improvements to the day-to-day development workflow.

I have also worked for BBC World Service, where I architected a cloud-based feed aggregation service and helped to build the BBC's 31 international responsive news websites. These websites all share the same codebase as BBC News, and are driven primarily by metaprogramming small generic presentation components.

Senior Developer at Rightster

December 2013 — August 2014

At Rightster I helped to architect and build the client-facing interface for the company's video distribution platform. We used Zend Framework 2 as a back-end & presentation layer, and made extensive use of AngularJS to build rich user interfaces.

As well as leading the JavaScript development, I helped to define a solid architecture and build process for the project's front-end. This was vitally important as the application contained over 20,000 lines of JavaScript. We used RequireJS and the AngularJS module system to decouple the code into small, easily-testable components.

Senior Developer at Time Inc. UK

May 2012 — December 2013

At Time Inc. I worked on building out a new API-driven mobile website platform. During this project I became interested in automated testing as a way to improve the quality of our products. I wrote a smoke test tool which uses PhantomJS to check for common page-level errors, and takes a screenshot of each page to aid manual visual checks. This helped catch bugs quickly, and enabled us to deploy the platform several times a day with confidence.

I also introduced JavaScript unit testing to the engineering department and encouraged more automated testing during the monthly developer gathering. The slides from my talk are available here.

Developer at Chroma

April 2011 — May 2012

At Chroma I was involved in designing and building an interface for bookmakers to monitor and set pricing in real time. The real-time aspect of this system was achieved using long polling, which allowed us to support legacy web browsers.

I also used the Campaign Monitor API to build a module for the ExpressionEngine CMS which allowed editors to create and manage their email campaigns from a familiar interface.

Developer at Zeevo

June 2010 — April 2011

Using the CodeIgniter framework and Doctrine ORM, I helped to build a booking and scheduling system used by tourism operators around New Zealand.

I was also involved in designing and building a secure file sharing application using the Dropbox API.
