Hello!

I'm a web developer for BBC News. I want to build web sites for the next billion, so unsurprisingly I have a passion for performance and accessibility. Most days you can find me writing JavaScript, Ruby, and Shell. When I'm not building for the web, I'm trying to improve workflows by creating tooling and optimising build pipelines.

In my spare time I maintain several open source projects, and I like to write Haskell and Elm. Outside of programming I am a cook, runner, rock climber and guitarist. I'm also an accidental keyboard enthusiast. I sometimes blog at Wildly Inaccurate.

Experience

Principal Developer at BBC News

August 2014 — Present

I am currently the development lead for the BBC News front page rebuild project. Built with Node.js, the front page is composed from React components which are shared through an internal open source system. These components are so far in use by BBC Sport, BBC Weather, and BBC Live.

Previous to the front page project, I lead a small team in redesigning the BBC News build pipeline using Jenkins and Docker. I have developed and still maintain several workflow and productivity tools used by developers in BBC News. I have also worked for BBC World Service where I assisted in the rollout of dozens of responsive web sites, and helped to architect & build the BBC's new cloud-based feed aggregation service.

Senior Developer at Rightster

December 2013 — August 2014

At Rightster I helped to architect and build the front-end for the company's video distribution platform. We used Zend Framework 2 as a domain logic & presentation layer, and made extensive use of AngularJS to build rich user interfaces.

The project consisted of over 20,000 lines of JavaScript, which we managed by adhering to a solid architecture around small, well-tested components.

Senior Developer at Time Inc. UK

May 2012 — December 2013

At Time Inc. I worked on building out a new API-driven mobile web site platform. During this project I became interested in automated testing as a way to improve the quality of our products. I wrote a smoke test tool which uses PhantomJS to check for common page-level errors, and takes a screenshot of each page to aid manual visual checks. This helped catch bugs quickly, and enabled us to deploy the platform several times a day with confidence.

I also introduced JavaScript unit testing to the engineering department and encouraged more automated testing during the monthly developer gathering. The slides from my talk are available here.

Developer at Chroma

April 2011 — May 2012

Working in a small close-knit team I atchitected and built a web interface used by bookmakers around the UK to monitor and set pricing in real time. To support legacy browsers, I chose to implement a long polling mechanism to achieve the "real time" functionality.

I also worked closely with Chroma's clients to create bespoke modules for the ExpressionEngine CMS, including an interface for managing email campaigns using the Campaign Monitor API, and a membership management tool for Racing UK Club.

Developer at Zeevo

June 2010 — April 2011

Using the CodeIgniter PHP framework and Doctrine ORM, I helped to build a booking and scheduling system used by tourism operators around New Zealand.

I was also involved in architecting and building a secure file sharing application using the Dropbox API.
